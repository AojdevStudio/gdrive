<objective>
Fix the Calendar `getEvent` cache key to include `calendarId`, preventing cache collisions when the same event ID exists across multiple calendars. Update all invalidation sites to use the corrected key format.

Purpose: The current cache key `calendar:getEvent:${eventId}` omits `calendarId`, which means looking up eventId "abc" on calendar A could return a cached result from calendar B if both have an event with that ID. This is a correctness bug.

Output: `getEvent` cache key becomes `calendar:getEvent:${calendarId}:${eventId}`; `updateEvent` and `deleteEvent` invalidation keys updated to match; existing test assertion tightened to verify the full key format.
</objective>
